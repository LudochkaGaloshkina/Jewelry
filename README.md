# Jewelry
<div align="center">💎 Сайт ювелирных изделий – Diamond Blackstar</div><br>

📌 Описание проекта <br>
 &nbsp;&nbsp;&nbsp;&nbsp; **Diamond Blackstar** — это веб-приложение для онлайн-просмотра и покупки ювелирных изделий премиум-класса. Сайт предоставляет пользователям удобный каталог украшений с возможностью детального просмотра характеристик изделий. Платформа ориентирована на современный дизайн, интуитивную навигацию и комфортный процесс выбора украшений. Проект направлен на создание стильного и функционального интернет-магазина ювелирной продукции.

🎯 Целевая аудитория: <br>
  &nbsp;&nbsp;&nbsp;&nbsp; Мужчины и женщины в возрасте от 18 лет <br>
  &nbsp;&nbsp;&nbsp;&nbsp; Люди, ищущие украшения для себя или в подарок<br>
  &nbsp;&nbsp;&nbsp;&nbsp; Покупатели, предпочитающие онлайн-шопинг<br>
  &nbsp;&nbsp;&nbsp;&nbsp; Ценители эксклюзивных и премиальных ювелирных изделий<br>

⚙ Ключевой функционал<br>
&nbsp;&nbsp;&nbsp;&nbsp;	📂 Просмотр каталога ювелирных изделий<br>
&nbsp;&nbsp;&nbsp;&nbsp;  🔍 Фильтрация и поиск товаров<br>
&nbsp;&nbsp;&nbsp;&nbsp;  📄 Просмотр детальной информации о товаре<br>
&nbsp;&nbsp;&nbsp;&nbsp;  🛒 Добавление товаров в корзину<br>
&nbsp;&nbsp;&nbsp;&nbsp;  ❤️ Добавление в избранное<br>
&nbsp;&nbsp;&nbsp;&nbsp;  👤 Регистрация и авторизация пользователей<br>
&nbsp;&nbsp;&nbsp;&nbsp; 💳 Оформление заказа<br>



<br><div align="center">❗ Основные разделы сайта</div><br>

| Название раздела  | Описание раздела |
| ------------- | ------------- |
| Главня        | Основная страница с баннерами, новинками и акциями. (/) |
| Аунтефикация  | Регистрация / Вход / Восстановление пароля (/auth/*) |

| Избранное  | Избранные товары пользователя (/favourites) |
| Контакты  | Форма обратной связи, адрес, телефон, социальные сети. (/contact) |
| Админ-панель  | Раздел доступный только пользователям с ролью администратор, позволяющая управлять учетными записями (/admin)  |

**🛠Технологический стек**<br>

🖥 Frontend
Vue.js
Почему выбран Vue.js:
1. Компонентная архитектура — позволяет разбивать интерфейс на переиспользуемые компоненты (карточка товара, шапка, футер, фильтры), что упрощает поддержку и масштабирование проекта.

2. Низкий порог входа — Vue проще в освоении по сравнению с некоторыми альтернативами, что ускоряет разработку.

3. Реактивность — автоматическое обновление интерфейса при изменении данных (например, динамическое обновление корзины или каталога).

4. Производительность — виртуальный DOM обеспечивает быструю работу даже при динамических списках товаров.

5. Поддержка SPA — идеально подходит для создания одностраничного приложения интернет-магазина.
&nbsp;&nbsp;Для сайта ювелирных изделий это важно, так как требуется динамический каталог, фильтрация, поиск и обновление интерфейса без перезагрузки страницы.

<br>
Vue Router

Почему выбран Vue Router:
1. Официальный роутер для Vue.js — полностью совместим с экосистемой.

2. Поддержка динамических маршрутов (например, /catalog/:id).

3. Возможность создания приватных маршрутов (например, /profile, /admin).

4. Удобная организация структуры SPA без перезагрузки страницы.

5. Это позволяет реализовать логичную структуру сайта: главная, каталог, карточка товара, личный кабинет и админ-панель.
<br>
🛠 Backend<br>
Node.js

Почему выбран Node.js:
1. Использование JavaScript и на frontend, и на backend — единая экосистема разработки.
2. Высокая производительность при работе с асинхронными запросами (поиск товаров, авторизация, работа с БД).
3. Подходит для REST API.
4. Большое количество библиотек и готовых решений.

> Для интернет-магазина это важно, так как сервер должен быстро обрабатывать запросы пользователей и взаимодействовать с базой данных.
<br>

Express.js<br>
Почему выбран Express.js:
1. Лёгкий и минималистичный фреймворк для Node.js.
2. Удобная маршрутизация (например, /api/catalog, /api/auth).
3. Поддержка middleware (аутентификация, логирование, обработка ошибок).
4. Простота создания REST API.

> Express позволяет быстро реализовать серверную логику: регистрацию, авторизацию, работу с товарами и заказами.

<br>
MySQL<br>

Почему выбрана MySQL:
1. Реляционная структура подходит для интернет-магазина (таблицы: пользователи, товары, заказы, категории).

2. Поддержка связей между таблицами (например, пользователь → заказы).

3. Надёжность и стабильность.

4. Поддержка SQL-запросов для сложной фильтрации и поиска.

5. Широкое распространение и хорошая документация.

> Для ювелирного магазина важно хранить структурированные данные о товарах, характеристиках и заказах.

<br>
🌐 Инфраструктура<br>
Локальный хостинг (Frontend + Backend + База данных)<br>
В рамках разработки проекта Diamond Blackstar вся инфраструктура разворачивается локально на компьютере разработчика.
<br><br>
📌 Frontend (локальный сервер разработки)<br>

Frontend-приложение на Vue.js запускается через встроенный сервер разработки (например, Vite).

**Обоснование выбора локального размещения:**
1. Удобство разработки и тестирования без публикации в интернет.

2. Быстрая перезагрузка (Hot Reload) при изменении кода.

3. Отсутствие затрат на внешний хостинг.

4. Достаточно для учебного проекта или курсовой работы.

> Frontend доступен по адресу типа: http://localhost:5173

<br><br>
📌 Backend (локальный сервер Node.js + Express<br>
Серверная часть запускается локально через Node.js.<br>
Обоснование:
1. Простота настройки и запуска (node server.js).

2. Полный контроль над API во время разработки.

3. Удобная отладка через консоль и инструменты разработчика.

4. Не требуется аренда удалённого сервера.

> Backend доступен по адресу типа: http://localhost:3000

<br><br>
📌 База данных (локальная MySQL)<br>
База данных MySQL устанавливается и работает локально (например, через MySQL Server или XAMPP).<br>
Обоснование:
1. Быстрое подключение к backend без сетевых ограничений.

2. Простота администрирования (phpMyAdmin / MySQL Workbench).

3. Безопасность — база данных недоступна извне.

4. Достаточно для демонстрации функционала интернет-магазина.

5. Не требует дополнительных затрат.
